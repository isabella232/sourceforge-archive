Virtual Hosts


Jetty supports the virtual host mechanism of HTTP. The methods to add a WebApplication or a HttpContext all take an optional virtualhost argument. In the jetty.xml file, this can be done with: 
      <Call name="addWebApplication">
    <Arg>virtual.host.name</arg>
    <Arg>/context</Arg>
    <Arg><SystemProperty name="jetty.home" default="."/> "/webapps/myapp"</Arg>
    <Arg><SystemProperty name="jetty.home" default="."/> "/etc/webdefaults.xml" </Arg>
    <Arg type="boolean">false</Arg>
  </Call>
If you want more than one virtual name for the webapp, you can add aliases with 
      <Call name="addHostAlias">
   <Arg>virtual.host.name</arg>
   <Arg>othername</arg>
  </Call>

  <Call name="addHostAlias">
   <Arg>virtual.host.name</arg>
   <Arg>192.168.0.1</arg>
  </Call>
