Jetty Architecture
These are just notes. A nice annotated diagram is on its way...

  • JVM contains one or more HttpServer instances.

  • HttpServer contains one or more HttpListener instances.
  • HttpServer contains one or more HandlerContext instances organized as an ordered list at each context path.
  • HttpServer contains zero or more UserRealm instances.

  • HandlerContext is registered at one and only one context pattern (eg. /context/*).
  • HandlerContext contains one or more HttpHandler instances organized in the order they were added.

  • ResourceHandler is a HttpHandler that serves files.
  • ServletHandler is a HttpHandler that serves configured Servlets by mapping a pathspec within the context (eg *.xxx, / or /XX/* ) to a ServletHolder
  • DynamicHandler is a ServletHandler that uses class names in URLs to configure servlets.
  • SecurityHandler is a HttpHandler that is mapped to UserRealm by name.

  • A WebApplication is a HandlerContext that uses a web.xml file to initialize a ResourceHandler and a ServletHandler.

  • A ServletContext is mapped to the HandlerContext, thus context initParams and attributes are in a context scope.
